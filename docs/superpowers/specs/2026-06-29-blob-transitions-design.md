# Blob System v2: Transitions & Architecture

## Summary

Upgrade the decorative blob background system to smoothly transition between pages, fix bottom-of-page clipping, and improve internal architecture. The blob shapes and waggle behavior stay the same; this is a structural and animation overhaul.

## Goals

1. **Cross-page transitions (primary):** When navigating between pages, old blobs drift-and-fade out toward their counterpart new blob positions while new blobs drift-and-fade in from those directions, implying a hand-off. Each set keeps its own page color (no cross-fade).
2. **Z-depth wafting:** Incoming blobs scale up from ~0.92 and deblur (4px -> 0); outgoing blobs scale down and blur out. Implies depth — blobs rising from behind the page.
3. **Bottom overflow fix:** Blobs that extend past page content are no longer clipped.
4. **Single animation loop:** Replace 7 independent RAF callbacks with one shared loop.
5. **Flat data model:** Blob state as plain config objects, generated by pure functions, rendered by simple components.

## Non-goals

- Changing blob shapes (separate follow-up)
- Scroll-driven interactions
- Smoothing internal curves
- Responsive blob count (keep at 7; can revisit later)

## Architecture

### Data types

```ts
interface BlobConfig {
  position: { top: number; left: number }; // rem, %
  shapePoints: BlobPath; // start, control, end (from BlobPathGeometry)
  waggleOffsets: BlobPath; // random phase offsets for waggle
  color: string; // hex color for this blob's stroke
}

interface BlobSetState {
  configs: BlobConfig[];
  seed: string;
  phase: "entering" | "visible" | "exiting" | "gone";
  startTime: number; // when this phase began
  driftVectors?: { dx: number; dy: number }[]; // per-blob drift direction, set during transitions
  staggerDelays?: number[]; // per-blob random delay (0-150ms), seeded
}
```

### Pure functions

**`generateBlobConfigs(seed: string, color: string): BlobConfig[]`**
Takes a seed string and blob color. Creates a `DeterministicVendor`, generates 7 blob configs with positions (top: 2-12rem, left: 10-90%), shape points, and waggle offsets. Pure, no side effects.

**`computeDriftVectors(oldConfigs: BlobConfig[], newConfigs: BlobConfig[]): { dx: number; dy: number }[]`**
For each paired blob (by index), computes a unit-ish direction vector from old position to new position. The drift distance is ~30-50% of the full vector — enough to imply direction without traversing the whole distance.

**`computeWagglePoint(point: Point, offset: Point, time: number, waggleSize: number, wagglePeriod: number): Point`**
Pure function replacing `BlobWaggler.waggledPoint()`. Given a base point, phase offset, current time, and waggle parameters, returns the waggled position.

**`computeBlobPath(config: BlobConfig, time: number, waggleSize: number): string`**
Computes the full SVG path string for one blob at a given time. Calls `computeWagglePoint` for each of the 3 points, then `svgCurvePath`.

### Components

```
BlobLayer.svelte          — persistent in layout, never destroyed
  BlobSet.svelte          — a group of 7 blobs with shared transition state
    BlobRenderer.svelte   — one SVG blob, receives pre-computed path string
```

**BlobLayer.svelte**

- Lives in `+layout.svelte`, receives `seed` (from `$page.url.pathname`) and `colorScheme` (from `$page.data`)
- Maintains a reactive list of `BlobSetState` objects (1 during steady state, 2 during transitions)
- Watches for seed changes via `$effect`
- On seed change:
  1. Generates new `BlobConfig[]` from new seed
  2. Computes drift vectors between old and new configs
  3. Sets old set's phase to `'exiting'` with drift vectors pointing toward new positions
  4. Creates new set in `'entering'` phase with drift vectors pointing from old positions
  5. After transition duration, removes exited set and sets new set to `'visible'`
- Runs a single `requestAnimationFrame` loop for waggle only:
  - Computes waggled path strings for all visible blobs across all sets
  - Writes to a shared state array that BlobRenderer components read reactively
  - Waggle runs continuously; transition animations are handled separately by CSS transitions in BlobSet
- Respects `prefers-reduced-motion`: skips waggle, makes transitions instant (opacity only, no drift/blur/scale)

**BlobSet.svelte**

- Receives: `BlobConfig[]`, phase, drift vectors, stagger delays
- Applies CSS for the current phase:
  - `entering`: starts at `opacity: 0; transform: scale(0.92); filter: blur(4px)`, transitions to `opacity: 0.4; transform: scale(1); filter: blur(0)`, with per-blob drift from offset to final position
  - `visible`: steady state, `opacity: 0.4`
  - `exiting`: transitions to `opacity: 0; transform: scale(0.92); filter: blur(4px)`, with per-blob drift toward counterpart
- Per-blob stagger via `transition-delay`
- Fires a callback when the exit transition completes (via `transitionend` event)

**BlobRenderer.svelte**

- Receives: SVG path string, stroke width, size
- Renders a single `<svg>` with a `<path>`
- No animation logic — purely presentational
- Color set via inherited CSS custom property or direct prop

## Overflow fix

Current BlobLayer CSS:

```css
position: absolute;
width: 100%;
top: 0;
bottom: 0;
z-index: -1;
overflow-y: hidden; /* this clips blobs */
```

New BlobLayer CSS:

```css
position: absolute;
width: 100%;
top: 0;
z-index: -1;
overflow-x: clip; /* prevent horizontal scrollbar from edge blobs */
pointer-events: none; /* explicit, was implicit via z-index */
```

Remove `bottom: 0` so the container grows to fit its children. Remove `overflow-y: hidden` so blobs extending past page content are visible. Add `overflow-x: clip` to prevent horizontal scrollbar. The blobs will naturally extend into the empty space below the page footer.

## Transition timing

All values are starting points — expect tuning during implementation.

| Parameter                 | Value                   | Notes                                           |
| ------------------------- | ----------------------- | ----------------------------------------------- |
| Total transition duration | ~800ms                  | From seed change to old set removed             |
| Per-blob stagger range    | 0-150ms                 | Random delay per blob, seeded for determinism   |
| Drift distance            | 30-50% of paired vector | Enough to imply direction, not a full traversal |
| Scale range               | 0.92 - 1.0              | Subtle depth cue                                |
| Blur range                | 0 - 4px                 | Softens appearance/disappearance                |
| Opacity range             | 0 - 0.4                 | Matches current steady-state opacity            |
| Easing (drift)            | ease-in-out             |                                                 |
| Easing (fade/blur/scale)  | ease-out                |                                                 |

## Reduced motion

When `prefers-reduced-motion: reduce`:

- Waggle animation disabled (existing behavior)
- Page transitions use opacity-only crossfade (no drift, blur, or scale)
- Transition duration shortened to ~200ms

## Color handling

- Each `BlobSet` receives its color as a prop, not from CSS custom properties
- During transitions, two sets coexist with different colors — old set keeps old page color, new set uses new page color
- The `--color-light-blob` / `--color-dark-blob` CSS custom properties remain on the page div for any non-blob uses, updating immediately on navigation
- Dark/light mode: `BlobSet` receives both light and dark colors, uses `prefers-color-scheme` media query to pick which to apply (same as current approach, just prop-driven instead of CSS-variable-driven)

## What stays the same

- `DeterministicVendor` — same xorshift PRNG seeded from URL pathname
- Blob shape — quadratic bezier with thick stroke (stroke-width = size/2)
- Waggle math — sinusoidal offsets with 2.5s period
- 7 blobs per set
- Position ranges — top: 2-12rem, left: 10-90%
- SVG size — 600px
- `BlobPathGeometry.ts` and `SVGPath.ts` — unchanged
